# nonebot-plugin-image-symmetry 插件智能体开发规范

## 代码质量标准

### 代码检查与格式化
- **使用 Ruff** 进行代码检查：提交前运行 `/path/to/venv/bin/ruff check`
- Ruff 检查后**不应出现任何错误、警告或信息提示**
- 保持代码整洁，避免死代码和不必要的复杂性

### 类型注解
- 类型提示中使用显式的 `Optional[T]` 而非隐式的 `T = None`
- 示例：使用 `img_bytes: Optional[bytes] = None` 而非 `img_bytes: bytes = None`

### 代码组织
- 将重复逻辑提取为辅助函数以提高可维护性
- 将导入语句移至文件顶部；避免在函数内部导入
- 当字典或常量被多次使用时，提取为模块级常量

### 错误处理
- 尽可能使用具体的异常类型而非捕获裸 `Exception`
- 确保所有异常都使用 nonebot 日志记录器进行适当记录
- 发生错误时返回 `None` 或适当的回退值，而非抛出异常

### 文档
- 使用中文编写文档字符串，因为这是一个中文项目
- 保持文档字符串简洁且信息丰富
- 确保文档字符串中的空行不包含空白字符
